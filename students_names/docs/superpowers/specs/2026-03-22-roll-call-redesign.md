# 快速点名器重新设计规范

**日期**: 2026-03-22
**状态**: 已批准
**类型**: 完整重新设计

## 概述

重新开发课堂点名器系统，核心目标是打造一个**快速点名工具**，采用轻量动画、加权随机、本地存储和可编辑名单。

## 整体架构

### 单文件架构

单个 `index.html` 文件包含所有 HTML、CSS、JavaScript 代码。无需构建工具，开箱即用。

### 模块分层

```
┌─────────────────────────────────────────┐
│              UI Layer                   │
│  (DOM 操作, 事件处理, 界面更新)           │
├─────────────────────────────────────────┤
│           Service Layer                 │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ Picker  │ │ Student │ │ Record  │   │
│  │ Service │ │ Service │ │ Service │   │
│  └─────────┘ └─────────┘ └─────────┘   │
├─────────────────────────────────────────┤
│            Data Layer                   │
│  ┌─────────┐ ┌─────────┐               │

│  │   DB    │ │ Storage │               │
│  └─────────┘ └─────────┘               │
└─────────────────────────────────────────┘
```

### 轻量动画

使用 CSS `transform: rotate()` 配合渐变圆形，替代 Three.js 3D 地球。

- 正常状态：缓慢旋转
- 点名中：加速旋转
- 显示结果：停止旋转

### 交互流程

1. 点击"开始点名" → 圆圈加速旋转 → 减速停止 → 显示姓名
2. 点击"正常/迟到/缺勤"按钮记录状态
3. 自动保存到本地存储

## 核心服务

### PickerService（点名服务）

```javascript
// 加权随机算法
calculateWeight(student) {
    baseWeight = 1
    lateWeight = 1 + (lateCount × 0.5)
    return Math.min(lateWeight, 5)  // 上限5
}
```

- 根据迟到次数计算权重
- 迟到越多，被点中概率越高
- 权重上限为 5

### StudentService（学生服务）

- CRUD 操作（增删改查）
- 迟到计数管理
- 内置初始名单（20个姓名）

### RecordService（记录服务）

- 添加点名记录
- 按日期筛选记录
- 导出 CSV

### StorageService（存储服务）

- IndexedDB 优先，localStorage 降级
- 统一的数据访问接口

## 数据模型

### Student（学生）

```typescript
{
  id: string,        // UUID
  name: string,      // 姓名
  lateCount: number, // 迟到次数
  createdAt: number  // 创建时间戳
}
```

### Record（记录）

```typescript
{
  id: string,           // UUID
  studentId: string,    // 学生ID
  studentName: string,  // 学生姓名（冗余，用于查询）
  status: 'normal' | 'late' | 'absent',
  timestamp: number     // 时间戳
}
```

### 存储结构

- IndexedDB: `ClassRollCallDB` database
  - `students` store (key: id)
  - `records` store (key: id, index: timestamp)

## 状态管理

### 应用状态

```typescript
state = {
  phase: 'IDLE' | 'SPINNING' | 'SHOWING' | 'SELECTING',
  selectedStudent: Student | null,
  students: Student[],
  todayRecords: Record[]
}
```

### 状态转换

```
IDLE ─(点击开始)─> SPINNING ─(2秒)─> SHOWING
     ↑                                  │
     └────────(记录完成)────────────────┘
                    │
                    └──> SELECTING (等待选择状态)
```

### UI 同步

状态变化自动更新界面，确保 UI 与数据一致。

## 技术决策

| 项目 | 选择 | 原因 |
|------|------|------|
| 动画 | CSS Keyframes | 轻量，无需额外库 |
| 图标 | Emoji 或 SVG | 无外部依赖 |
| 存储 | IndexedDB + localStorage 降级 | 兼容性好 |
| 构建 | 无 | 单文件直接运行 |
| 浏览器支持 | Chrome 90+, Firefox 88+, Safari 14+ | 现代 API 支持 |

## 功能需求

### 核心功能

1. **点名功能**
   - 点击按钮开始点名

   - 加权随机选择

2. **状态记录**
