# 课堂点名器/签到系统 设计文档

**日期**: 2026-03-21
**项目**: Class Roll Call System

## 1. 项目概述

一个基于浏览器的课堂点名器，使用 3D 地球旋转动画作为视觉效果，支持学生名单导入、点名记录、迟到权重管理等功能。

**使用场景**: 课堂环境，老师在教室电脑上使用，投影显示

## 2. 技术选型

| 技术 | 用途 |
|------|------|
| HTML5 | 页面结构 |
| CSS3 | 样式和动画 |
| Vanilla JavaScript | 业务逻辑 |
| Three.js | 3D 地球渲染 |
| IndexedDB | 数据持久化 |

**实现方式**: 混合方案 - 单个 HTML 文件，内部代码按功能模块组织

## 3. 功能需求

### 3.1 核心功能
1. **CSV 导入学生名单**
2. **点名记录** - 记录每次点名的时间、学生、状态
3. **迟到权重系统** - 迟到次数越多，被点中概率越高
4. **CSV 导出记录** - 支持按时间范围导出
5. **3D 地球动画** - 持续旋转作为背景效果

### 3.2 点名流程
```
1. 老师点击"开始点名"
2. 系统根据权重随机选择学生
3. 地球加速旋转（视觉反馈）
4. 显示学生姓名（动画放大）
5. 弹出状态选择：正常 / 迟到 / 缺勤
6. 保存记录
7. 更新该学生迟到计数（如果迟到）
8. 更新今日统计
```

## 4. 数据结构

### 4.1 学生数据 (Student)
```javascript
{
  id: string,           // UUID
  name: string,         // 学生姓名
  lateCount: number,    // 迟到次数
  lastCalled: number    // 最后被点名时间戳
}
```

### 4.2 点名记录 (Record)
```javascript
{
  id: string,           // UUID
  studentName: string,  // 学生姓名
  timestamp: number,    // 时间戳
  status: 'normal' | 'late' | 'absent'
}
```

## 5. IndexedDB 数据库

```
数据库名: ClassRollCallDB
版本: 1

对象存储 'students':
  - keyPath: 'id'
  - indexes: name, lateCount

对象存储 'records':
  - keyPath: 'id'
  - indexes: timestamp, studentName
```

## 6. 加权随机算法

```
基础权重 = 1
迟到权重 = 1 + (lateCount × 0.5)

学生被选中概率 = 该学生权重 / 所有权重之和
```

**示例**:
- 学生 A：迟到 0 次 → 权重 1
- 学生 B：迟到 2 次 → 权重 2
- 学生 C：迟到 4 次 → 权重 3

总权重 = 6，学生 C 有 50% 被选中概率。

## 10. 核心模块

| 模块 | 职责 |
|------|------|
| **DataStore** | IndexedDB 封装，管理学生数据和记录 |
| **EarthRenderer** | Three.js 3D 地球渲染 |
| **RandomPicker** | 加权随机选择算法 |
| **UIController** | 界面交互控制 |
| **CSVHandler** | CSV 导入导出 |

## 11. 错误处理

- 无学生数据：提示"请先导入学生名单"
- CSV 解析失败：显示具体错误信息
- IndexedDB 不可用：降级使用 localStorage
