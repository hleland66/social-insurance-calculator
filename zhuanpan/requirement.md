自定义转盘应用 产品需求文档 (PRD)
产品概述
一款支持高度自定义、纯前端运行的轻量级转盘应用。用户可自由配置转盘选项与权重，切换极简治愈系主题，并在网页端获得沉浸式的抽奖体验。其核心特色在于支持将包含所有配置与逻辑的完整应用一键打包为单个离线 HTML 文件，实现跨设备、免安装的永久留存与二次编辑。
用户路径与核心场景
1 场景设计
用户面临日常选择困难（如决定午餐吃什么、周末去哪玩、任务分配等），需要一个具有视觉美感且操作低门槛的随机工具。
2 核心路径
打开页面进入配置视图 -> 输入转盘标题 -> 增删选项并分配权重数值 -> 挑选视觉主题 -> 切换至全屏游玩模式 -> 点击抽奖获取反馈 -> 导出单文件至本地保存。
功能模块详细需求
配置控制台视图
1 全局标题设置
提供一个居中显示的大号输入框，默认提示文案为“今天决定点什么？”等占位符。用户输入的标题将实时同步展示在游玩视图的正上方或转盘中心。
支持最大输入字数限制（建议 20 字以内），超出部分自动截断或禁止输入。
2 选项与权重管理列表
列表区提供清晰的表单行。每行包含：选项名称输入框（限制 15 字）、权重数值输入框（仅限正整数或至多一位小数，不接受负数与零）、单条删除按钮。
默认状态下提供 3 到 5 个占位选项，权重默认为 1。
列表底部固定添加新选项按钮。当选项达到上限（建议最多 20 个）时，按钮置灰不可点击。
提供一键清空功能，并配有二次确认弹窗，防止用户误触导致数据丢失。
3 权重比例自动换算机制
系统后台自动计算所有输入数值的总和，并按比例换算为每个选项占 360 度的具体弧度，实时反映在预览转盘的扇形面积划分上。这极大降低了用户的认知负担，无需强行拼凑 100%。
若所有选项权重均为空或 0，界面需给出非侵入式的错误提示，此时禁止进入游玩模式或触发导出。
4 主题画廊选择器
以横向滑动的色卡或缩略图形式展示预设主题。
预设主题需涵盖高阶质感与治愈系风格，例如莫兰迪色系、极简留白风格或轻拟物风格。
点击任意主题卡片，整个页面的背景色、转盘扇形配色组合、指针样式、按钮 UI 及绑定的反馈音效将进行全局无缝切换。
游玩沉浸视图
1 沉浸式状态切换
提供开始游玩主操作按钮。点击后，配置表单平滑淡出或滑出屏幕边缘，转盘主体与标题放大并移动至视觉正中心。
提供隐蔽但易于触达的返回编辑入口（如页面角落的悬浮齿轮图标），点击后平滑恢复表单显示。
2 核心物理转动逻辑
用户点击转盘中心的抽奖按钮或页面底部的触发区开始转动。
转动过程必须包含三个物理阶段：初始爆发加速（持续 0.5 秒）、高速匀速旋转（持续 1-2 秒）、带有自然阻尼感的平滑减速（持续 2-3 秒）。
目标结果在点击瞬间由后台计算完成，前端动画需精准匹配计算出的圈数与最终停留角度，确保指针绝对指向预定结果扇形的中央区域，严禁视觉与底层逻辑错位。
转动期间，所有可能干扰状态的点击交互（包含再次点击开始、返回编辑、导出等）均需被锁定。
3 结果反馈与重新开始
转盘完全停止的瞬间，触发高亮视觉动效（如目标扇形轻微放大或周边浮现光晕）及清脆的提示音。
屏幕中央或转盘上方弹出结果展示面板，包含放大的中奖选项名称。
结果面板提供再转一次按钮，点击后关闭弹窗，转盘复位或直接从当前位置准备下一次转动。
单文件离线打包机制
1 导出触发与数据收集
用户在编辑态或游玩态均可点击导出为 HTML 按钮。
系统需将当前页面存在的选项名称列表、对应的权重数值、以及最后选定的主题标识，打包提取为一份 JSON 格式的状态快照。
2 资源内嵌与零依赖限制
打包引擎需将应用运行所需的 HTML 骨架、CSS 样式（含极简主题特有的渲染规则）、JS 运行逻辑拼接成一个长字符串。
所有必须的视觉资源（如指针 SVG 代码）和音频文件（如转动音效）必须预先转换为 Base64 编码内嵌于代码中，确保生成的 HTML 文件在完全断网的环境下依然能完美渲染和发声。
3 离线文件读取与唤醒
用户下载的本地 HTML 文件双击在浏览器打开后，默认直接进入游玩沉浸视图状态。
初始化机制需优先读取代码内部预埋的状态快照来渲染转盘，直接呈现用户导出前的最终样貌。
本地文件同样保留悬浮的配置入口，点击唤出表单。用户在本地修改参数后，可以再次点击自带的导出按钮，生成更新后的新版本文件。
交互与体验指标
1 极简治愈美学
去除一切冗余的边框和生硬的阴影，采用大圆角和通透的排版设计。
避免使用高饱和度的刺眼色彩，转盘相邻扇形的颜色对比需柔和过渡。
所有状态切换、弹窗显隐均需包含至少 0.2 秒的缓动过渡动画，杜绝界面的瞬间闪跳。
2 听觉反馈设计
提供匹配主题的极简音效。例如轻拨齿轮的白噪音、木鱼敲击声或清脆的铃音。需提供全局静音开关。
3 响应式适配
页面需完全适配移动端与桌面端屏幕。在移动端竖屏状态下，配置表单与转盘呈上下流式布局；在桌面端横屏状态下，呈左右分栏或画中画布局，确保空间利用率最大化。
