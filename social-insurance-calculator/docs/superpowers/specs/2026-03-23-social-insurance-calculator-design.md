# 五险一金计算器设计文档

## 项目概述

构建一个基于 Next.js + Supabase 的轻量级 Web 应用，用于计算员工社保缴费基数和公司缴费金额。

**定位：** MVP（最小可用产品），可扩展为 SaaS 工具

**目标用户：** 小微企业主/HR

## 技术栈

- **前端框架：** Next.js (App Router)
- **样式：** Tailwind CSS
- **后端/数据库：** Supabase (PostgreSQL)
- **数据解析：** xlsx (SheetJS)
- **部署：** Vercel

## 数据库设计

### cities（城市社保标准表）

```sql
create table cities (
  id bigint generated by default as identity primary key,
  city_name text not null,
  year text not null,
  base_min integer not null,
  base_max integer not null,
  rate numeric not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  unique(city_name, year)
);
```

### salaries（员工工资表）

```sql
create table salaries (
  id bigint generated by default as identity primary key,
  employee_id text not null,
  employee_name text not null,
  month text not null,
  salary_amount integer not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  unique(employee_id, month)
);
```

### results（计算结果表）

```sql
create table results (
  id bigint generated by default as identity primary key,
  employee_id text not null,
  employee_name text not null,
  city_name text not null,
  year text not null,
  avg_salary numeric not null,
  contribution_base numeric not null,
  company_fee numeric not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);
```

## 核心业务逻辑

### 计算流程

1. 用户选择要计算的城市（从已上传的 cities 表中选择）
2. 从 salaries 表读取所有员工数据
3. 按 employee_id 分组
4. 根据所选城市的 year 过滤对应年份数据
5. 计算年度月平均工资：`avg_salary = 总和 / 月份数`
6. 获取所选城市的标准（base_min, base_max, rate）
7. 计算缴费基数：
   - avg_salary < base_min → base_min
   - avg_salary > base_max → base_max
   - 否则 → avg_salary
8. 计算公司缴费：`company_fee = contribution_base × rate`
9. 清空 results 表，写入新结果（关联所选城市）

### 数据规则

- 所有金额保留 2 位小数（四舍五入）
- 每次计算前清空 results 表

## API 设计

### POST /api/upload

解析 Excel 并写入数据库。

**请求：** FormData with file

**响应：**
```typescript
{
  success: true,
  data: {
    citiesInserted: number,
    salariesInserted: number
  }
}
```

### GET /api/cities

获取可用的城市列表（用于页面下拉框）。

**响应：**
```typescript
{
  success: true,
  data: {
    cities: Array<{
      city_name: string;
      year: string;
    }>
  }
}
```

### POST /api/calculate

执行社保计算。

**请求：**
```typescript
{
  city_name: string;
  year: string;
}
```

**响应：**
```typescript
{
  success: true,
  data: {
    calculated: number
  }
}
```

### GET /api/results

获取计算结果。

**响应：**
```typescript
{
  success: true,
  data: {
    results: Array<{
      employee_id: string;
      employee_name: string;
      city_name: string;
      year: string;
      avg_salary: number;
      contribution_base: number;
      company_fee: number;
    }>
  }
}
```

## 页面设计

### 首页 `/`

- 居中标题："五险一金计算器"
- 两个大按钮垂直排列（商务蓝配色）
- 点击跳转对应页面

### 上传页面 `/upload`

- 文件上传区（拖拽 + 点击选择）
- 城市选择下拉框（从数据库中的 cities 表加载，上传成功后自动更新）
- 两个操作按钮：[上传数据]、[执行计算]
- 状态提示：loading、成功、失败

### 结果页面 `/results`

- 表格展示计算结果
- 表格列：员工编号、姓名、城市、年份、月均工资、缴费基数、公司缴费
- 状态：加载中、空状态、有数据

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| Excel 格式错误 | 提示"请上传正确的 Excel 文件" |
| 缺少 Sheet | 提示"Excel 必须包含 cities 和 salaries 两个 Sheet" |
| 找不到城市标准 | 提示"未找到 {city_name} 的社保标准数据" |
| 数据库连接失败 | 提示"数据库连接失败，请检查网络" |
| API 超时 | 提示"请求超时，请重试" |
